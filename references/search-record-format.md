# 搜索记录存档格式

每轮搜索结束后，将搜索记录保存为 `.md` 文件，存放在运行目录的 `search-records/` 子目录下（如目录不存在则先创建）。

## 文件命名

```
主题-YYMMDD.md
```

- `主题`: 简洁的中文主题标签，无空格（如需分隔用连字符），如 `新能源汽车市场`、`AI芯片供应链`
- `YYMMDD`: 搜索日期，如 `260323`
- 示例: `新能源汽车市场-260323.md`

如果同一主题之前搜索过，创建新的独立文件并在 `related` 字段中链接之前的文件——不要覆盖。

## 文件结构

```markdown
---
topic: 主题
date: YYYY-MM-DD
queries:
  - "检索词1（中文）"
  - "English query 2 filetype:pdf"
scope: 地域 | 时间范围 | 来源偏好
related:
  - 主题-260310.md   # 同主题上次搜索，留空则删除此字段
---

## 来源列表

| 标题 | 机构 | 日期 | 类型 | 访问 | 链接 |
|------|------|------|------|------|------|
| 标题文字 | 机构名 | 2024-Q3 | consulting report | official PDF | [官方页](URL) · [PDF](URL) |

访问标签: `official PDF` / `official page` / `wechat official account` / `authorized distribution` / `secondary mirror`
类型标签: `consulting report` / `white paper` / `broker research` / `regulator/exchange` / `company IR` / `association/standard` / `wechat official account` / `platform research`

## 主要内容摘要

**强制要求：来源列表中的每一条信源都必须有对应的摘要块，不得遗漏任何一条。**

每条来源写一个摘要块：

### [标题](URL)

- **目录/结构**: 列出报告的主要章节或部分，如 "第1章 市场规模 · 第2章 竞争格局 · 第3章 政策环境 · 附录 数据表"
- **核心内容**: 一句话说明这份资料解决了什么问题
- **检索提示**: 后续写报告时，哪个主题应查此文件的哪个章节，如 "市场规模数据→第2章；政策梳理→第4章"

## 亮点数据

列出值得直接引用的具体数字、统计数据或发现。使用项目符号格式：

- **数据点**: 具体数字或结论 | 来源: [机构名](URL) | 页码/章节（如知道）

## 资料覆盖评估

- **偏向类型**: 本次搜索以哪类资料为主（如 咨询报告为主，缺乏监管/官方数据）
- **明显缺口**: 哪个角度或数据点没有找到权威来源

## 缺口与受限来源

列出存在但无法完全获取的来源：

- [来源名称](落地页 URL) | 原因: 无公开 PDF / 已删除 / 链接失效 | 备注
```

保存规则：
- 写入前如 `search-records/` 目录不存在则先创建
- 不要修改或摘要化超出搜索返回内容的来源——记录是忠实归档，不是编辑改写
- 没有内容的章节可省略（如无缺口，省略"缺口与受限来源"）
- 文件保持自包含：一个没有看过对话的读者应能理解搜索了什么、找到了什么
