# 四种文档类型的骨架模板

## 1. 调研报告

适用场景：了解竞对/行业/品类怎么做的，给销售或业务决策参考。

```
## 一、Fact Sheet
  - 全量关键数据列出（消耗规模、增速、单位经济、客户分层、供给侧增量来源等）
  - 口径备注在此处简要说明

## 二、[核心议题]是怎么发生的/怎么涨起来的
  - 小标题全部用问句："XX发生了什么"、"竞对的关键动作是什么"、"客户分层怎么做的"
  - 每个板块讲完后跟一段"快手对照"

## 三、增量从哪来 / 客开怎么做的
  - 供给侧增量来源（量化）
  - 客开执行方式
  - 激励政策
  - 产品侧降低门槛的动作
  - 快手对照

## 四、[其他品类/细分]怎么涨的
  - 与核心议题的差异对照表
  - SKU角色分工
  - 快手对照

## 五、快手该怎么做
  - 每条按"原文支撑 → 在快手怎么用"两段表述
  - 优先动作（无技术依赖）/ 复盘动作（需内部协同）/ 慎重方向（边界条件）/ 硬约束（天花板测算）

## 六、待办与口径
  - 待办按依赖关系排序
  - 口径存疑集中列表

## 附录：参考来源
  - 外部素材链接 + 内部文档链接 + 写作风格参考链接
```

## 2. 访谈纪要整理

适用场景：多份访谈纪要需要整理成可读文档，归纳核心洞察。

```
## 一、访谈背景
  - 被访者、时间、场景、访谈目的

## 二、核心洞察（跨被访者归纳）
  - 所有被访者共识的 3-5 条判断
  - 每条标注"几位被访者中谁说了什么"

## 三、每位被访者的要点
  - 每人一个子板块，用问句小标题
  - 原话引用 + 结构化归纳

## 四、共识与分歧
  - 哪些问题大家看法一致
  - 哪些问题存在分歧，分歧点在哪

## 五、待验证假设
  - 从访谈中提炼出还没被验证的假设
  - 标注来源

## 附录：原始素材链接
```

## 3. 业务复盘

适用场景：某个业务动作/项目/季度结束后回顾。

```
## 一、目标回顾
  - 原定目标是什么（北极星指标 + 关键里程碑）

## 二、实际结果
  - 数据呈现（与目标对比）
  - 超预期 / 未达预期的部分

## 三、拆解归因
  - 小标题用问句："为什么超预期"、"为什么没达预期"、"哪些变量起了关键作用"
  - 每个归因追溯到具体动作/决策/外部因素

## 四、做对了什么 / 没做对什么
  - 可复制的经验（带具体动作描述）
  - 需要避坑的地方

## 五、下一阶段动作
  - 按依赖关系排序

## 附录：数据来源 + 原始材料链接
```

## 4. 方案/提案

适用场景：推动某个项目立项或方向决策。

```
## 一、要解决什么问题
  - 问题定义 + 影响范围 + 不解决的后果

## 二、方案概述
  - 一段话讲清楚核心思路
  - 为什么选这个方向（排除其他方向的理由）

## 三、详细设计
  - 按模块/阶段拆开，每个模块用问句小标题
  - 资源需求 / 排期 / 关键依赖

## 四、风险与预案
  - 每个风险：场景 + 影响 + 预案

## 五、ROI 估算
  - 投入（人力/时间/资金）vs 预期产出
  - 估算口径说明

## 附录：数据来源 + 参考材料
```
