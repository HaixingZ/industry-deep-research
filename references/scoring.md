# 评分

## 报告类型决定权重

评审前必须先读取 Research Brief 头部的"报告类型"字段。**不同类型权重不同**，用错权重 = 用错评审标准。

### 默认权重（立项材料型 / 决策备忘型）

```text
Total Score =
  0.20 × Coverage
+ 0.18 × Logic
+ 0.20 × Insight
+ 0.12 × Expression
+ 0.10 × Style Fit
+ 0.20 × Reality Check
```

### 方向汇报型权重（Direction Memo）

```text
Total Score =
  0.22 × Coverage
+ 0.20 × Logic
+ 0.25 × Insight
+ 0.13 × Expression
+ 0.10 × Style Fit
+ 0.10 × Reality Check（仅限"真问题判断"和"竞对辩证"两项；MVP/Kill/成功标准/停止条件不参与扣分）
```

> ⚠️ **方向汇报型不强求 MVP 路径、Kill Criteria、成功标准阈值、时间表 / 投入估算**——这些属于立项材料的内容，必须先和相关方对齐过才能写。在汇报型报告里写未对齐的具体动作和数字，是扣分项而非加分项。如果报告里出现这类内容但 Brief 标注为方向汇报型，评审 agent 应判断"建议挪到立项材料 / 删除拍脑袋数字"。

### 学习/调研型权重（Knowledge Build）

```text
Total Score =
  0.30 × Coverage
+ 0.20 × Logic
+ 0.15 × Insight
+ 0.15 × Expression
+ 0.10 × Style Fit
+ 0.10 × Reality Check（仅限"真问题判断"；其他不强求）
```

每个维度按 10 分制评分。通过阈值：**9.0/10**。

## 维度定义与 0-10 分标准

### 1. Coverage（覆盖度）权重 0.20

- 报告回答了必答问题
- 主要章节有实质性展开
- 草稿读起来像一份完整报告，而非一个框架
- 证据基础对该主题足够广泛，没有人为缩窄

| 分数 | 标准 |
|------|------|
| 0-2 | 草稿是骨架或大纲，主要章节缺失或空壳 |
| 3-4 | 覆盖了部分必答问题，但多个章节缺乏实质展开，来源基础薄弱 |
| 5-6 | 回答了大部分必答问题，主要章节有展开，但仍有明显遗漏（如缺少关键竞对、缺少关键数据来源类别） |
| 7-8 | 全面覆盖必答问题，每个章节有实质内容，来源基础广泛，仅有个别边缘问题未涉及 |
| 9 | 完整覆盖所有必答问题，证据基础充分，来源组合合理（公司自述+外部分析），无人为缩窄 |
| 10 | 在9分基础上，主动发现并覆盖了用户brief未提及但逻辑上应该回答的问题 |

### 2. Logic（逻辑）权重 0.18

- 论证内在一致
- 结论没有超越证据支撑的范围
- 章节之间的推理逻辑连贯

| 分数 | 标准 |
|------|------|
| 0-2 | 论证混乱，结论与证据脱节，章节之间无逻辑关联 |
| 3-4 | 部分论证自洽，但有明显跳跃或矛盾未解决 |
| 5-6 | 主线逻辑清晰，但部分章节推理薄弱或存在内部矛盾 |
| 7-8 | 论证内在一致，章节间推理连贯，结论未超越证据支撑范围 |
| 9 | 论证严密，每个判断都有证据链支撑，显式标注了假设和不确定性 |
| 10 | 在9分基础上，识别并解决了读者可能的反驳（"有人会说X，但我们的回应是Y"） |

### 3. Insight（洞察）权重 0.20

- 报告有清晰的 thesis（核心命题）而非"全面覆盖"
- 每个 pattern 形成"现象→逻辑→So What"的完整闭环
- 提炼出跨研究对象（公司/平台/产品）的结构性规律，而非停留在单个对象的描述
- 有原创的分析概念或框架（不是生搬硬套已有理论）
- So What 具体可操作（"应该做什么"而非"值得关注"）
- 模式间有因果关系分析（哪些是硬约束、哪些是衍生现象）
- 至少有 1 个反共识判断（行业共识是什么？有没有证据表明共识可能是错的？）
- 附录中的事实细节服务于正文论点（不是为完整而完整）

| 分数 | 标准 |
|------|------|
| 0-2 | 纯粹facts堆砌，无任何模式提炼或规律识别 |
| 3-4 | 识别了一些区分或趋势，但停留在表面描述（"A做了X，B做了Y"） |
| 5-6 | 有跨对象的模式提炼，但So What薄弱或泛泛（"值得关注""需要跟进"） |
| 7-8 | 模式形成"现象→逻辑→So What"完整闭环，有原创概念，So What具体可操作 |
| 9 | 多个模式之间有因果关系网络，有反共识判断且有证据支撑，insight改变读者对该主题的认知框架 |
| 10 | 在9分基础上，insight具有预测性（"基于这个规律，预计X会发生"）且预测逻辑自洽 |

### 4. Expression（表达）权重 0.12

- 行文可读且结构清晰
- 冗余得到控制
- 定义和过渡清晰

| 分数 | 标准 |
|------|------|
| 0-2 | 不可读，结构混乱，大量冗余或矛盾 |
| 3-4 | 可读但粗糙，过渡生硬，定义不清 |
| 5-6 | 行文可读，结构清晰，但冗余未控制好或过渡不够流畅 |
| 7-8 | 行文流畅，结构清晰，冗余受控，数据呈现方式得当 |
| 9 | 表达精准，每段都有信息增量，开头钩子强，结尾有判断力 |
| 10 | 在9分基础上，文字有节奏感和画面感，复杂概念用类比/对比一次讲清 |

### 5. Style Fit（风格匹配）权重 0.10

- 符合用户风格规则
- 保持冷静、有用、适合讨论的语调
- 避免无依据的确定性或推销性语言

| 分数 | 标准 |
|------|------|
| 0-2 | 风格严重偏离（如用户要冷静分析但报告像营销文案） |
| 3-4 | 风格基本正确但有明显违和（如过度确定性、推销性语言） |
| 5-6 | 风格大致匹配，但有若干处语调不统一 |
| 7-8 | 风格匹配良好，保持冷静、有用、适合讨论的语调 |
| 9 | 精准匹配用户风格偏好（如有reference text则句式节奏、用词偏好、段落组织均一致） |
| 10 | 在9分基础上，风格自然到"像用户自己写的"程度 |

### 6. Reality Check（逍姐视角）权重 0.20

用逍姐的思维框架审视报告的战略建议和商业判断。核心信念："我们唯一的动机是要赚钱。"

> ⚠️ **Reality Check 按报告类型分级适用**：
>
> - **立项材料型 / 决策备忘型**：6 个追问全部强制，权重 0.20
> - **方向汇报型**：仅"真问题判断"和"竞对辩证"两项强制，权重 0.10。MVP / 成功标准 / 停止条件 / 商业本质 / 可行性这五项**不强求**，未对齐过的具体动作和数字反而是扣分项
> - **学习/调研型**：仅"真问题判断"一项参考，权重 0.10

| 分数 | 标准 |
|------|------|
| 0-2 | 建议全是"战略价值""值得关注"，零商业本质检查，零可行性评估 |
| 3-4 | 提到了商业价值但没有具体回答"能赚钱吗"，可行性停留在"需要评估" |
| 5-6 | 部分建议回答了商业本质，但成功标准模糊，无MVP路径，无停止条件 |
| 7-8 | 每个建议都回答了"能赚钱吗"，有成功标准和MVP路径，区分了"竞对在做"和"我们应该做" |
| 9 | 商业本质清晰（具体收入来源和规模估算），成功标准可验证（定量+基线+及格线），有明确停止条件，有"不做的事"清单，有MVP路径（最痛客户是谁） |
| 10 | 在9分基础上，主动识别了报告中的"假问题"（"这个焦虑可能是行业炒作"），台面话和实际逻辑分开说，考虑了组织内部约束（团队/预算/优先级冲突） |

**Reality Check 的 6 个追问**（评审必须逐项检查；标 ⭐ 项对方向汇报型同样强制）：

1. ⭐ **真问题 vs 假问题**："这到底是不是真问题？" — 报告的核心焦虑有证据链支撑吗？还是行业在炒作？
2. ⭐ **竞对辩证**："竞对做了 ≠ 我们也要做" — 建议是否检查了先行条件？我们的场景变量有没有？
3. **商业本质**："能赚钱吗？" — 每个建议是否回答了具体收入来源，而非只说"战略价值"（**仅立项/决策型强制**）
4. **可行性**："能不能干成？" — 有团队、预算、时间线、停止条件吗？还是"长期想法太多了干不过来"？（**仅立项/决策型强制**）
5. **成功标准**："什么样算好？说不清说明没想明白" — 每个建议是否有可验证的成功标准（定量+基线+及格线）？（**仅立项/决策型强制**）
6. **MVP路径**："找个最痛的客户先验证" — 有没有最小可行验证路径？还是"三个月做完美产品=死路一条"？（**仅立项/决策型强制**）

> ⚠️ **方向汇报型的反向扣分规则**：如果 Brief 标注为方向汇报型，但报告里出现了 MVP 路径、Kill Criteria、具体阈值（命中率 X%、误操作 < Y 元）、时间表（P0/P1/P2 + 投入工程师月）等内容，且没有证据表明这些已和相关方对齐过——评审 agent 应在评语里指出"这些属于立项材料的内容，未对齐过会被老板第一句话追问数字怎么来的"，并建议删除或挪到立项材料中。这是逍姐讲过的"拍脑袋的内容宁可不写，也不要待补占位"的硬规矩。

## 关卡

这些关卡是通过/不通过的。不要将它们隐藏在数值评分中。

### 完整报告关卡

仅当草稿是真正的长篇报告时才通过。
如果是骨架式、简略的或主要是大纲文本，则不通过。

### 证据卫生关卡

仅在以下条件满足时通过：
- 证据/假设/缺口被清晰区分
- 未支撑的论断没有被写成事实
- 证据局限性是可见的，没有被隐藏

### 引用链接关卡

仅当依赖来源的段落使用了可点击的 Markdown 引用（`[标题](URL)`）时才通过。
如果 source-dependent 段落仅提及报告、文章或来源名称而未附加链接，则不通过。裸名称不通过。

### Brief 对齐关卡

仅当草稿实质性地回应了已批准的 `Research Brief`，特别是其中声明的 `Report Goal` 和 `Must-Answer Questions` 时才通过。
如果起草者或评审者悄然缩窄论点，然后按照那个更窄的版本来评判报告而未获得用户审批，则不通过。

### 深度研究来源组合关卡

仅在以下条件满足时才通过——对于行业、市场、竞争、渠道或广告支出论断，且更广泛的外部来源合理存在时：
- 草稿不是主要基于公司财报构建的
- 主要的跨公司或全市场论断背后至少有一类非公司分析来源的支持，或者缺失的来源类别已被明确记录为已搜索但不可用或受限

### 来源充分性关卡

仅当每个主要章节对当前置信度水平有足够的支撑材料时才通过。
如果报告因来源基础薄弱或不完整而内容单薄，则不通过。

### 逍姐关卡（Reality Check Gate）

> **按报告类型分级判定**——见上方 Reality Check 维度的"⚠️ 分级适用"说明。

**立项材料型 / 决策备忘型**（全条强制，仅当所有条件满足时通过）：
- 每个战略建议都回答了"能赚钱吗"（具体收入来源，不是"战略价值"）
- 报告的核心焦虑有证据链支撑（不是"行业趋势"一句话带过）
- 建议区分了"竞对在做"和"我们应该做"，并检查了先行条件
- 至少有 1 个建议包含可验证的成功标准（定量指标+基线+及格线）和停止条件
- 没有"三个月做完美产品"式的建议——必须有 MVP 路径（最痛的客户是谁，怎么伴随测试）

**方向汇报型**（仅前两条强制，仅当以下条件满足时通过）：
- 报告的核心焦虑有证据链支撑（不是"行业趋势"一句话带过）
- 横向竞品分析后给出了"竞对在做 ≠ 我们也要做"的辩证判断，并落到自家具体项目（不是只给"参考蓝图"）
- **反向检查**：报告里**没有**未对齐过的 MVP 路径 / 具体阈值 / 时间表 / Kill Criteria——如果有，关卡不通过，需要删除或挪到立项材料

**学习/调研型**（最低门槛）：
- 报告的研究范围和概念定义清晰，无明显误导性事实

## 路由逻辑

- `pass`
  当总分 >= 9.0 且所有关卡均通过时使用。

- `rewrite`
  当报告不缺信息但写法有问题（结构、逻辑、洞察、表达、风格或 Reality Check 不达标）时使用。不是 improve，是重写对应章节。触发条件：关卡通过或接近通过，但某个维度 < 8 分。

- `search`
  当 Brief 对齐关卡、来源充分性关卡或深度研究来源组合关卡未通过，且公开来源可能仍然存在时使用。也适用于 Reality Check 维度因缺少商业数据（收入、成本、团队规模）而得分低的情况。

- `rewrite+search`
  同时需要重写和补充搜索时使用。

- `blocked`
  当 Brief 对齐关卡、来源充分性关卡或深度研究来源组合关卡未通过，主要原因是所需材料无法通过公开渠道获取时使用。

## 平台期逻辑

当以下条件成立时，运行处于平台期：
- 连续 2 轮评审每轮总分提升 < 0.7 分
- 剩余弱点主要受限于信息获取阻塞（而非写作问题）

当平台期出现时，停止迭代并清晰记录阻塞原因。
