# 深度行业研究：一个人人都可以拥有的 AI 战略分析师

> 它不是「帮我搜一下」的搜索引擎，也不是「帮我写一篇」的文案工具。它是一个会自动找高质量信源、自动反复修改直到达到专业标准的 AI 战略分析师。装在你的 myflicker 里，随叫随到。

---

## 为什么做这个？

我用了很久的 AI 写报告，踩过几乎所有的坑。

让 ChatGPT 写一份行业研究报告，体感是：前两段读起来像麦肯锡，翻到第三页开始编数据，点开引用链接——404。表面上是一份报告，骨子里是一篇文笔不错的综述，经不起推敲。

更深层的问题是：**写报告和找资料是同一个脑子。** 它自己搜、自己写、自己评——既当运动员又当裁判。缺少资料的时候，它不会说「我找不到」，而是靠文笔硬撑过去。

我需要的东西其实很明确：

1. **它会认真地找资料**，不是随便 Google 一下就开始写
2. **它会自己审自己**，不是写完就算完
3. **它会反复改**，直到真的达标，不是我觉得差不多了就停

所以我做了这个 —— 一个跑在 myflicker 里的研究报告 skill。它把「找资料」和「写报告」拆成两个独立的 Agent，用文件通信，按标准评审，自动迭代。

---

## 它是怎么工作的？

核心架构是一个双人系统：

```
┌──────────────────────────┐      ┌──────────────────────────┐
│   规划/评审 Agent          │      │   执行 Agent              │
│                          │      │                          │
│   · 写 Research Brief     │ ───► │   · 按优先级搜索           │
│   · 写 Search Plan        │      │   · 验证每一条来源         │
│   · 审核 Storyline        │ ◄─── │   · 全力写报告            │
│   · 评审 + 打分 + 路由    │ ───► │   · 根据反馈修改           │
│                          │      │   · 补搜 → 再改           │
└──────────────────────────┘      └──────────────────────────┘
         ▲
         │  "这篇不行，竞争格局那章
         │   只有公司年报，没有第三方视角。
         │   去搜券商的公开研报和公众号。"
```

**挑刺的 Agent 和干活的 Agent 是两个人。** 挑刺的那位在第一步就跟你确认「这份报告要回答什么问题」，然后它制定搜索计划。干活的出去搜、回来写。挑刺的再上场——打分、查关卡、直说哪里弱。如果不过关，打回去改。可能还要补搜。补搜完了再改。循环。

直到挑刺的那位说「过了」，或者确认「公开能找的资料已经到天花板了」，才会停下来。

## 搜索：不是 Google 一下，是按优先级穷尽

很多 AI 写报告的第一步是「搜索」，但只是把关键词丢进搜索引擎拿前几条结果。

这个 skill 的搜索逻辑是这样的：

| 优先级 | 信源类型 | 说明 |
|:---:|------|------|
| 1 | 官方 PDF | 出版方直接托管的原文，不是转载 |
| 2 | 官方页面 | 没有 PDF 但有落地页，比如监管机构公告 |
| 3 | 微信公众号 | 券商研报、咨询报告、行业分析——很多首发渠道就是公众号 |
| 4 | 授权分发 | 公开数据库、活动页面 |
| 5 | 二次转载 | 最后手段，会明确标注风险 |

付费墙后面的内容直接跳过。如果搜了一圈，最好的资料是某个论坛的转载帖——它会诚实地告诉你「没找到更好的公开版本」，而不是假装它很权威。

搜索完之后，**每一条来源都要写摘要**：这份资料讲了什么、哪个章节有哪些数据、后续写报告时应该查它的哪一部分。这个归档动作看起来麻烦，但它保证了后续写报告的时候不是在「凭印象引用」。

## 写作：不是写完就算，是写到过关为止

每一版报告都要过两道关：

**第一道：5 维打分**

覆盖度 (28%) · 逻辑 (22%) · 洞察 (18%) · 表达 (16%) · 风格适配 (16%)

**第二道：6 个硬关卡**

1. **完整报告关** — 是真正的长篇报告，不是大纲充字数
2. **证据卫生关** — 证据 / 假设 / 缺口被清晰区分，没支撑的论断不装成事实
3. **引用链接关** — 每个引用都是可点击的链接，只写报告标题不给链接 = 不通过
4. **Brief 对齐关** — 报告有没有回答一开始确认的那些问题？偷偷缩小论题 = 不通过
5. **深度研究来源关** — 行业级论断不能全凭公司年报，必须有第三方视角
6. **来源充分性关** — 每个主要章节的支撑材料够不够？

打回来改。改完再审。审完可能还要补搜。

这套流程听起来很严苛，但正因为如此，产出的报告才不是那种「开头气势磅礴、中间数据模糊、结尾一堆空链接」的东西。

## 风格：不是只能写一种腔调

你注意到 5 维打分里有一个「风格适配」(Style Fit)，占了 16% 的权重。这个维度做的事是：**确保报告不是你指定的风格，就等着被打回来改。**

怎么指定风格？在第一步确认 Research Brief 的时候直接说就行。比如：

| 场景 | 你可以说 |
|------|---------|
| 咨询报告风 | 「写成麦肯锡式的咨询报告，用金字塔结构，每段结论先行」 |
| 券商研报风 | 「像券商行业深度研报，有投资逻辑、关键假设、风险提示」 |
| 学术论文风 | 「学术风格，引用规范，表格用三线表，避免口语化表达」 |
| 商业叙事风 | 「像《财经》杂志的长篇报道，有故事线，数据和叙事交替推进」 |
| 创始人简报 | 「写给投资人看的，一页结论 + 正文，直接、不废话、结论加粗」 |
| 产品经理调研 | 「结构化但不用太正式，可操作性强，每个结论后面给 action item」 |
| 政策分析风 | 「偏政府智库风格，政策沿革 + 现状分析 + 政策建议三段式」 |

你也可以说得更细——「多用短句」「不要起承转合直奔结论」「每章结尾给一句话小结」「数据全部放表格不要藏在段落里」。

**更进阶的用法：直接给它一篇你写的文章。** 如果你历史上写过类似的报告或分析文章，把那段文字贴到 Research Brief 里（或者给个链接），它会从五个维度拆解你的文风——句式节奏、用词偏好、段落组织、数据呈现、语气姿态——然后在写报告的时候对着复刻。你在评审环节也能看到 Style Fit 那一项是不是在「更像你」的方向上迭代。

评审 Agent 会照着你给的标准或样本逐版检查，偏了就拉回来。

**风格不只是「文笔好」**，它影响评审的路由决定。如果报告信息扎实但风格不对，评审会让它走 `improve` 路径（改写法，不补搜）。反之，风格对了但证据弱，会走 `search` 路径（去找资料，不改写法）。

本质上，这个系统把「写作风格」从玄学变成了可检查、可迭代的工程指标。

## 停止：不是一直改下去，是知道什么时候该停

有个硬性停止条件：**如果连续两轮评分提升不到 0.7 分，而且剩下的弱项主要因为公开资料已经找不到更多了——就停。**

这时候它会产出一份 `Evidence Gap Log`，诚实记录：哪些结论是可靠的，哪些是基于推断的，哪些还需要人去搞内部数据。

这可能是整个系统最被低估的设计决策：**知道什么时候说「我不知道」，比假装全知更重要。**

## 断点恢复：工作状态保存在文件里，随时可以捡起来

所有中间产物都写成 markdown 文件，存在 `report_runs/<主题>/` 目录下：

```
report_runs/新能源汽车充电桩-260517/
├── 01-research-brief.md       # 研究边界（跟你确认过）
├── 02-search-plan.md          # 搜索计划（跟你确认过）
├── search-records/            # 每一轮搜索的存档
├── 04-storyline-packet.md     # 叙事框架（跟你确认过）
├── 05-report-v1.md            # 报告第一版
├── 06-review-v1.md            # 评审意见
├── ...
├── final-report.md            # 最终报告
├── evidence-gap-log.md        # 证据缺口记录
└── run-status.md              # 断点恢复文件
```

关了电脑、换了会话、过了一周——读到 `run-status.md`，它就知道上次做到哪了。

**这个设计的另一个好处是：评审是可追溯的。** 每一轮的评审意见都留着，你能看到评审 Agent 每次提了什么意见、执行 Agent 改了什么。不是黑箱操作。

## 最后

这个项目的出发点很简单：**好的研究能力不应该被机构垄断。**

一个券商的策略分析师，背后是一个研究部在支撑它的信息来源。一个咨询公司的顾问，有内部的数据库和方法论。而大多数需要做行业研究的人——创业者、产品经理、投资人、记者、学生——只有一个搜索引擎和一堆 SEO 优化的垃圾内容。

这个 skill 试图把那个「研究部」的职能塞进一个 AI 系统里。不是替代人的判断，而是把「找资料、验证资料、组织资料、反复打磨」这些体力活自动化。

**让它做苦力。你做判断。**
